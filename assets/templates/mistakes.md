# 실수·교훈 누적

**중요 산출물을 확정하기 전 이 파일을 먼저 확인**하고, 같은 실수의 조짐이 보이면 즉시 항목을 추가한다.
항목 형식: **증상 → 원인 → 재발 방지책**. 번호(M-001, M-002, …)를 증가시키며 누적하고, 해결됐다고 삭제하지 않는다(교훈은 만료되지 않는다).

## M-000 — (형식 예시 — 첫 실제 교훈을 기록할 때 이 블록을 대체하라)

**증상:** (무엇이 잘못됐는가 — 관찰된 현상을 구체적으로. 언제·어디서·어떤 산출물에서)

**원인:** (근본 원인 — 표면 증상이 아니라 왜 발생했는가. 검증 누락·가정 오류·도구 함정 등)

**재발 방지책:** (다음에 어떻게 적용하는가 — "조심한다"가 아니라 실행 가능한 형태로: 확정 전 실행할 검증 명령, 체크리스트, 금지 패턴)
