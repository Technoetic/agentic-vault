세션 종료 프로토콜을 수행하라.

## 0. 볼트 감지 (필수 게이트)

현재 작업 디렉토리 루트에 `00-meta/vault-config.json`이 존재하는지 확인하라.
- **없으면**: "이 디렉토리는 agentic-vault 볼트가 아니므로 /vault-session-end를 생략합니다." 한 줄만 정중히 안내하고 즉시 종료하라.
- **있으면**: 설정을 읽어라. `handoff_note`, `hot_note`, `log_note`, `log_tags`, `backup_target` 키를 사용한다. 빈 문자열 키에 해당하는 단계는 조용히 생략한다.

## 1. handoff 갱신 (handoff_note 설정 시)

이번 세션에서 수행한 작업을 회고하여 `handoff_note` 파일을 **전체 갱신**하라 (500단어 이내):

- §1 최근 완료된 작업 / §2 인간의 확인이 필요한 사항 / §3 보류된 작업 / §4 다음 세션 지시사항
- 프런트매터 `updated`를 오늘 날짜로 갱신하라.
- 파일이 아직 없으면 위 4개 섹션 구조와 프런트매터(설정의 `required_keys` 준수)를 갖춰 새로 생성하라.

## 2. hot 컨텍스트 갱신 (hot_note 설정 시)

`hot_note`(볼트 핫 컨텍스트, 500단어 이내)의 "지금 상태"와 "최우선 미결" 섹션을 이번 세션 결과로 갱신하라.

## 3. log 기록 (log_note 설정 시)

`log_note` 최상단에 이번 세션 1줄 요약을 추가하라.

- 요약 앞에 설정의 `log_tags` 중 가장 적합한 연산 태그를 붙여라 (예: `[ops]`, `[build]`) — 로그를 grep 가능한 데이터로 유지하는 규약이다.
- `log_note` 헤더에 자체 형식 정의가 있으면 그 형식을 따르고, 없으면 `- YYYY-MM-DD HH:MM | 행위자 | [태그] 요약` 형태로 적어라.

## 4. 결정 기록 (해당 시)

이번 세션에서 아키텍처/전략 결정이 확정되었으면 볼트의 결정 기록(ADR) 노트에 추가하라. 위치가 불분명하면 `index_note`에서 찾아보고, 볼트에 결정 기록 관행이 없으면 생략하라.

## 5. 장기 기억 커밋 (선택 단계)

장기 기억 MCP 서버(예: `memoryhub`의 `remember` 류 도구)가 **연결되어 있으면** 핵심 결정·교훈·블로커를 커밋하라.

- **신뢰도 기반 캡처**: 향후 세션에 지속 영향을 줄 지식(신뢰도 50% 초과)만 선별하고, 일회성 세부사항·단순 수정·handoff와 겹치는 만료성 작업 상태는 제외하라.
- 3~5문장으로 압축하라.
- 연결되어 있지 않으면 조용히 생략하라 (보고 시 한 줄 언급으로 충분).

## 6. git 커밋 (볼트가 git 저장소인 경우)

볼트 루트가 git 저장소이면 이번 세션에서 변경한 노트를 커밋하라.

- `git add <변경파일>` 후 커밋 — 과잉 스테이징(`-A`)을 지양하고 이번 세션 변경분만 스테이징하라.
- `.gitignore`가 제외하는 바이너리·비밀 파일은 그대로 두라.
- **⚠️ 원격 push는 절대 금지** — 개인 지식 볼트에는 기밀·개인정보가 포함될 수 있다. push는 사용자가 명시적으로 지시할 때만, 그때도 기밀 포함 여부를 먼저 경고하라.
- git 저장소가 아니면 이 단계를 조용히 생략하라.

## 7. 백업 권고 (backup_target 설정 시)

설정에 `backup_target`이 비어 있지 않으면, 사용자에게 백업 스크립트 실행을 권고하라 (예: "backup_target이 설정되어 있습니다 — 백업 실행을 권장합니다"). 사용자 확인 없이 백업을 임의 실행하지는 마라.

## 8. 종료 보고

갱신한 파일 목록·커밋 여부·생략한 단계(사유 포함)를 사용자에게 간결히 요약 보고하라.
