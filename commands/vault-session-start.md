세션 복원 프로토콜을 수행하라.

## 0. 볼트 감지 (필수 게이트)

현재 작업 디렉토리 루트에 `00-meta/vault-config.json`이 존재하는지 확인하라.
- **없으면**: "이 디렉토리는 agentic-vault 볼트가 아니므로 /vault-session-start를 생략합니다." 한 줄만 정중히 안내하고 즉시 종료하라. 에러를 출력하거나 파일을 생성하지 마라.
- **있으면**: 그 파일을 읽어 설정 객체로 삼아라. 이하 모든 경로는 이 설정의 키(`handoff_note`, `hot_note`, `index_note`, `log_note` 등)를 참조한다. 키 값이 빈 문자열이면 해당 단계를 조용히 생략한다(우아한 성능 저하).

## 1. 상태 파일 로드

1. `handoff_note`가 설정되어 있고 파일이 존재하면 읽어라 (직전 세션 인수인계 — 가장 신선한 상태). 볼트가 git 저장소이고 handoff 상단에 **기준 커밋(anchor)** 해시가 있으면, 현재 HEAD와 다를 때 `git log --oneline <anchor>..HEAD`로 그 사이 커밋을 확인해 handoff에 없는 최신 변경을 브리핑에 반영하라.
2. `hot_note`가 설정되어 있으면 읽어라 (볼트 핫 컨텍스트 — 지금 상태·최우선 미결 요약).
3. `index_note`를 읽어 볼트의 전체 지도를 파악하라. 분량이 크면 최상단 구조 섹션만 훑어라.
4. handoff와 hot이 모두 비어 있으면 `log_note` 최상단 5~10줄로 최근 활동을 대신 파악하라.

## 2. 장기 기억 회상 (선택 단계)

장기 기억 MCP 서버(예: `memoryhub`의 `recall_context` 류 도구)가 **연결되어 있으면** 1회 호출해 세션 횡단 기억을 로드하라.
- 연결되어 있지 않거나 unavailable 응답이 오면 그 사실만 한 줄로 보고하고 계속 진행하라. 재시도하지 마라.
- 회상 내용이 볼트 파일과 모순되면 **볼트를 우선**하라 — 볼트가 진실의 원천(Ground Truth)이다.

## 3. 브리핑

다음 형식으로 간결히 브리핑하라 (읽은 파일에 해당 정보가 없는 항목은 "정보 없음"으로 표기):

- **볼트 상태**: (hot_note 또는 index_note 기준 1~2줄)
- **직전 세션 요약**: (handoff 기준. handoff 미설정 시 log 최근 항목 기준)
- **확인 필요**: (handoff의 "인간의 확인이 필요한 사항" 기준)
- **다음 작업 후보**: (handoff의 "다음 세션 지시사항" 또는 hot의 "최우선 미결" 기준)

## 4. 시작점 질문

사용자에게 어떤 항목부터 시작할지 물어라.
